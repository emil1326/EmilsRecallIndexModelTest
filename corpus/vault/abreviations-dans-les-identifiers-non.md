Les abréviations semblent pratiques sur le moment mais créent de l'ambiguïté plus tard. "EW" peut vouloir dire plein de choses, "EmilsWork" c'est non-ambigu. Dans les namespaces Unity genre EmilsWork.Editor, c'est clair à qui appartient ce code et dans quel contexte. Spell it out, toujours.
