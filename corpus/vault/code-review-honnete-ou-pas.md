Si le code a des issues de performance, de lisibilité, ou de design, faut le dire direct. Emil prend ça comme de l'information utile, pas comme une attaque personnelle. Un review qui approuve tout pour maintenir la paix crée de la dette technique déguisée, pis ça c'est plate tbh.
