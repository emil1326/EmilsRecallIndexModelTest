Un tutorial te tient la main pis te dit quoi typer, mais un side project te lance dans le vide pis c'est là que t'apprends vite. Les décisions de design, les edge cases weird, les bugs inattendus — t'as pas le choix de les dealer. Emil build ses tools par besoin réel, pis c'est exactement pour ça que ça marche.
