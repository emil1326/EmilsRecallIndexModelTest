La VS Code source control panel c'est utile pour voir le diff visuel, mais pour les commits, rebase, cherry-pick, CLI seulement. Git en terminal c'est verbose mais au moins tu comprends ce que tu fais, pas de magie cachée. GitLens dans VS Code c'est l'exception, pour voir le blame pis l'historique c'est harder to beat.
