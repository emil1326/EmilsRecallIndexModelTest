C'est la contrainte la plus simple du branding: C# compile pas si ton class name a une apostrophe. Alors Emils c'est la seule option pour les class names, pis c'est fine. Le display name dans l'éditeur peut avoir l'apostrophe, mais le class name reste propre. No drama.
