Pour les micro-décisions (nommer une variable, choisir un pattern local), juste faire pis mentionner c'est plus efficace que de tout approuver. Pour les décisions qui affectent comment EmilsWork se présente ou comment le système est structuré, consulter d'abord. Le coût d'une consultation préventive est bas, le coût d'un revert architectural est élevé.
