GitHub a le concept de Draft PR — tu peux ouvrir une PR marked as draft, ça signal aux reviewers que c'est pas prêt pour un full review mais que le contexte est là si ils veulent regarder. Moi j'ouvre souvent une draft dès que j'ai un premier proof of concept qui build, même si le code est rough. Ça évite le classique 'ah je pensais tu voulais faire autrement' après des heures de travail.
